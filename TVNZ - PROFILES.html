INTRODUCTION

The Problem

TVNZ OnDemand is a free video platform. They are using all resources to develop a new feature, creating profiles. This must benefit Sales, Marketing and Product by obtaining the name of user, year of birth and gender. My task was to design the interface for profiles on the big screen platforms (smart tvs, Playstation, Xbox)

01 RESEARCH THE CURRENT INTERFACE

Experience the current interface

Before begining any ideation, I wanted a hands-on experience of what it felt like navigating TVNZ OD with a remote on the television via a remote, playstation and xbox controller.

Current Interface Key Insights

Left aligned keyboard on search, FAQ and Categories

Settings placed on the top-nav of the screen navigation

Small screen real-estate on the right of the screen

BIG SCREEN INTERFACE

Requirements

Name, year of birth, gender and if a kid’s profile is required

Has to fit-in with the current UI

Has to be achieveable within the timeline and budget for the developer

02 DEFINING THE NEED AND IDEATE

Ideation

I needed to design an interface that enables users to create a profile in a fun and friendly way by only using a remote or controller. I came up with concepts utilising the current interface, and one “unique” idea to see if the developers were able to achieve change in the interface.

Concept One - Current UX

This concept stuck to the current keyboard design, being fixed to the left. The issue with this concept was the focus point would jump between the left and right of screen. Consistency is cruical with big screen User experience.

CONCEPT ONE

Concept Two - Dynamic Keyboard

This concept had a keyboard which would be collasped, untill the user selected a form to enter text. This would push the form to the right of screen, putting the focus state on the keyboard.

CONCEPT TWO

Concept Three - Step by Step

This concept broke the steps down into different steps on the right of screen. The keyboard would stay fixed, or show options relative to the steps.

CONCEPT THREE

Concept Four - Wizard

This concept was braching away from the current design of the keyboard being fixed on the left. I thought having the profile elements and keyboard centered, it would create a better user experience.

CONCEPT FOUR

03 DECIDING ON WHAT CONCEPT

Meeting

After the UX Manager picked the top five concepts, I then designed these into mid-fi prototypes, using Sketch. I held a meeting with the head of development, UX Manager, Porduct Manager, Senior Digital Designer, Business Analyists and Big Screen Platform Developer to decide on the top concept. The chosen idea was concept 04, wizard concept.

CHOSEN CONCEPT - MID-FI - WIZARD

Flow of the Experience

Once the concept was decided, required screens were written on a large sheet of paper, then the site-map was created.

05 HALLWAY TESTING

User Testing

I designed all of the screens in Sketch, and prototyped the design into an MVP. A testing problem I encountered was figuring out a way to test the prototype mimicking a remote. The quickest solution was to have a faux remote next to the screen. Each button would connect to another screen, showing where the focus state would be.

PROTOTYPE TESTING WITH FAUX REMOTE

User Testing Insights

Focus state on gender selection was confusing

Users spent most time finding the caps-lock upon entering their name

Users were confused on what to enter for ‘year of birth’

06 ENHANCING USER EXPERIENCE

Solving the Pain Points

Applying some creative problem-solving, the following solutions were applied the the problems:

FOCUS STATE

The current focus state didn’t stand out enough to show the user where they were currently focused. A UI design change was applied to ensure more contrast when selecting options in the Wizard.

YEAR OF BIRTH INPUT

A low opacity placeholder text was applied to prompt the user of what year they were born. Once the user had entered the fourth number, the focus state then automatically focuses on ‘Next’ to reduce click-count.

Name Input

The keyboard now automatically capitalises the first letter of the name, then automatically lower-cases the next letters to ensure consistency with profile names and less clicks.

07 FINAL DESIGN

Released on TVNZ OnDemand

Once the updates were made, the hi-fi designs were uploaded to Zeplin where the developer coded the interface through HTML, CSS and Javascript.
